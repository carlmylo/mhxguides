  <TabItem label="Windows">
* ¡REAPER me da `Error al abrir dispositivos` cuando cambio a WASAPI!
  > <Image src={errordevicewin} alt="REAPER mostrando un error que dice 'There was an error opening the audio hardware. WASAPI Error: could not find input device'." />
  * `No se pudo encontrar entrada`: Es muy probable que tienes un micrófono conectado o el formato no coincide con la de tu salida (i.e. altavoces están a 48,000 Hz y el micrófono esta a 44,100 Hz).  
  Si es la primera, cambia `Canales de entrada` a `0` en la sección de `Dispositivos` en las Preferencias.  
  Si es la segunda, cambia el formato para que coinciden juntos.
* ¡No me carga el plugin para visualizar (RBN Preview)!
  > <Image src={errorpluginwin} alt="REAPER mostrando un error con RBN Preview." />
  * No deberias de seguir usando este plugin, pero si quieres ser terco, te faltan archivos de Microsoft Visual C++ Redistributable Runtimes.  
  <a href="https://kutt.it/vcpp" target="_blank" rel="noopener noreferrer">[Haz click aquí para descargarlos]</a>.  
</TabItem>
<TabItem label="Arch">
* I get "`JACK: error creating client`" when swapping to JACK!
  > <Image src={errorjackarch} alt="REAPER showing an error that says 'There was an error opening the audio hardware. WASAPI Error: could not find input device'." />
  #### Solution A:
  * You have `jack2` installed and need to replace it with `pipewire-jack`.
    * Install `pipewire-jack` by running this command in a terminal.
        ```bash
        sudo pacman -S pipewire-jack
        ```
        > <Image src={archpwjack} alt="Una terminal descargando pipewire-jack de pacman." />
  #### Solution B:
  * Change to a different audio system, like `PulseAudio` or `ALSA`.
    > <Image src={settingsaudioarch} alt="REAPER's Preferences showing the Device section." />
</TabItem>

    > <Image src={settingsaudioarch} alt="REAPER's Preferences showing the Device section." />
